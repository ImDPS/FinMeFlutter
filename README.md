# FinMe

**Your money. Your phone. Your rules.**

A personal finance tracker built for India — because managing lakhs shouldn't cost lakhs. FinMe reads your bank SMS, crunches the numbers locally, and syncs to the cloud when *you* say so. No third-party servers touching your transactions. No subscriptions. Just you and your rupees.

---

## What's the Vibe?

```
 Dark Mode Only    |    Glassmorphism UI    |    INR Native    |    Encrypted Everything
```

FinMe is a **local-first**, **privacy-obsessed** finance app with a dark glassmorphism aesthetic. Your data lives encrypted on your device (SQLCipher), syncs to Firebase on your terms, and never leaves India's borders. Built with Flutter, powered by Riverpod, and designed for the way Indians actually bank.

## Getting Started

### Prerequisites

You'll need these installed and ready to roll:

- **Flutter** `>= 3.41.0` — [install guide](https://docs.flutter.dev/get-started/install)
- **Dart** `>= 3.3.0`
- **Java** `17` (for Android builds)
- **Android Studio** or **VS Code** with Flutter extension
- **Firebase CLI** — `npm install -g firebase-tools`
- **FlutterFire CLI** — `dart pub global activate flutterfire_cli`

### Clone & Setup

```bash
# 1. Clone the repo
git clone https://github.com/your-username/FinMeFlutter.git
cd FinMeFlutter

# 2. Install dependencies
flutter pub get

# 3. Generate code (Drift tables, Riverpod providers, GoRouter)
dart run build_runner build --delete-conflicting-outputs

# 4. Run the app
flutter run
```

> **Heads up:** Step 3 is mandatory after cloning. Any file with `@riverpod`, `@DriftDatabase`, `@DataClassName`, or `@UseRowClass` needs code generation. Generated files end in `.g.dart`.

### Firebase Setup

FinMe uses Firebase for auth (phone OTP) and cloud sync. You'll need your own Firebase project:

1. Create a Firebase project at [console.firebase.google.com](https://console.firebase.google.com)
2. Register an Android app with package name `com.finme.finme`
3. Add your debug SHA-1 key:
   ```bash
   cd android && ./gradlew signingReport
   ```
4. Download `google-services.json` -> place at `android/app/google-services.json`
5. Run FlutterFire configure:
   ```bash
   flutterfire configure --project=YOUR_PROJECT_ID
   ```
6. Enable **Phone Authentication** in Firebase Console
7. Create a **Firestore Database** (location: `asia-south1` for India)

For the full setup walkthrough, see [docs/FIREBASE_SETUP.md](docs/FIREBASE_SETUP.md).

### Environment Files You'll Need

These files are gitignored (they contain API keys) and must be created locally:

| File | How to Get It |
|------|---------------|
| `android/app/google-services.json` | Firebase Console -> Download |
| `lib/firebase_options.dart` | Auto-generated by `flutterfire configure` |

## How SMS Import Works

FinMe's killer feature. Here's the flow:

```
  Your Phone's SMS Inbox
         |
         v
  [Telephony Plugin]  -- reads bank SMS with permission
         |
         v
  [SmsParserService]   -- regex magic for HDFC, ICICI, Axis, Kotak, SBI...
   |  extracts: amount, type (debit/credit), account, merchant, bank
         |
         v
  [SmsImportRepository] -- SHA-256 dedup (no duplicate imports)
         |
         v
  [Drift Database]      -- encrypted local storage
         |
         v
  [SyncQueue]           -- queued for Firestore sync
```

Supports Indian number formats (`1,23,456.00`), handles OTP/promo/balance SMS rejection, and deduplicates with SHA-256 hashing. Your SMS is parsed **entirely on-device** — nothing leaves your phone.

## Architecture Deep Dive

### State Management — Riverpod 3.x

Every feature uses `@riverpod` annotation with code generation. State is **always immutable** with `copyWith()`.

```dart
// Provider generation: DashboardNotifier -> dashboardProvider
// (NOT dashboardNotifierProvider — Riverpod v3 drops the "Notifier" suffix)

@riverpod
class Dashboard extends _$Dashboard {
  @override
  DashboardState build() { ... }
}
```

### Database — Drift + SQLCipher

- **Production**: SQLCipher-encrypted, key stored in `FlutterSecureStorage`
- **Tests**: `AppDatabase.forTesting()` — in-memory, no encryption, no platform channels
- **6 tables**: accounts, transactions, budgets, net_worth_snapshots, sync_queue, user_settings

### Sync Engine

```
Local Write -> Drift DB (instant)
           |-> SyncQueue (enqueued)
                   |
                   v (every 24hrs or manual flush)
              Firestore
              (retry x3, then mark failed)
```

### Navigation — GoRouter 14.x

```
Auth Flow:    /onboarding -> /otp -> /lock
Main App:     /dashboard | /transactions | /budgets | /net-worth | /settings
Standalone:   /sms-import
```

## Testing

Tests mirror the `lib/` structure under `test/`. Real database, no mocks for data layer:

```dart
setUp(() {
  db = AppDatabase.forTesting();  // In-memory, fast, isolated
  repo = TransactionsRepository(db: db);
});
tearDown(() async => db.close());
```

Use `mocktail` for Firebase and other external services.

```bash
# Run everything
flutter test

# With coverage report
flutter test --coverage
# Coverage gate: >= 60% lines
```
